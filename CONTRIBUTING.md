# Contributing

Corrections, reproducibility improvements, and carefully scoped experiments are welcome.

## Before opening a pull request

```bash
python scripts/check_repository.py
```

For code changes that affect training, also describe:

- dataset and split construction;
- train, validation, and test transforms;
- seed, environment, and hardware;
- checkpoint-selection rule;
- exact commands and configuration; and
- machine-readable metrics or checksums produced by the run.

Do not present best-over-test accuracy as an unbiased held-out result. Do not call the custom SSM implementation an official S4/S4D reproduction. Never commit CIFAR-10 files, credentials, model caches, checkpoints without an explicit maintainer decision, or generated bytecode.

Keep changes focused and use an imperative commit subject. If you correct a recorded number, update the relevant report and explain which notebook or new artifact supports it.
